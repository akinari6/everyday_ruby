問題5-2: attr_*の使い分け
課題内容
商品を管理する Product クラスを実装し、Rubyの attr_reader、attr_writer、attr_accessor を使ってコードを簡潔にする方法を学びましょう。
要件
以下の仕様で Product クラスを実装してください：

属性の定義

name（商品名）: 読み取り専用（getter のみ）→ attr_reader を使う
price（価格）: 読み取り専用（getter のみ）→ attr_reader を使う
stock（在庫数）: 読み書き可能（getter と setter）→ attr_accessor を使う
description（説明）: 書き込み専用（setter のみ）→ attr_writer を使う


初期化メソッド

引数: name、price、stock（初期在庫数、デフォルトは0）
キーワード引数を使う


インスタンスメソッド

sell(quantity): 在庫を減らす

在庫が足りる場合: 在庫を減らして true を返す
在庫が足りない場合: 何もせず false を返す


restock(quantity): 在庫を増やす
available?: 在庫が1以上なら true



テストケース
ruby# Product クラスのインスタンス作成
product1 = Product.new(name: "ノートPC", price: 120000, stock: 5)
product2 = Product.new(name: "マウス", price: 2500)  # stock のデフォルトは 0

# attr_reader のテスト（読み取り可能）
p product1.name
# => "ノートPC"

p product1.price
# => 120000

# attr_accessor のテスト（読み書き可能）
p product1.stock
# => 5

product1.stock = 10
p product1.stock
# => 10

# attr_writer のテスト（書き込みのみ）
product1.description = "高性能なノートパソコン"
# OK（エラーにならない）

# p product1.description
# => NoMethodError（読み取りメソッドがないのでエラー）

# sell メソッドのテスト
p product1.sell(3)
# => true

p product1.stock
# => 7  (10 - 3)

p product1.sell(10)
# => false  (在庫不足)

p product1.stock
# => 7  (変わらない)

# restock メソッドのテスト
product1.restock(5)
p product1.stock
# => 12  (7 + 5)

# available? メソッドのテスト
p product1.available?
# => true

p product2.available?
# => false  (stock が 0)
ヒント

attr_reader: 読み取り専用（getter のみ）

ruby  attr_reader :name, :price
  # これは以下と同じ
  # def name
  #   @name
  # end
  # def price
  #   @price
  # end

attr_writer: 書き込み専用（setter のみ）

ruby  attr_writer :description

attr_accessor: 読み書き両方（getter と setter）

ruby  attr_accessor :stock

キーワード引数のデフォルト値: def method(arg: default_value)
インスタンス変数への代入: @stock += quantity や @stock -= quantity
