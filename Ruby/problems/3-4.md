問題3-4: メソッドチェーンの実践
課題内容
注文データから、複数の条件を組み合わせて情報を抽出・集計するメソッドを実装してください。メソッドチェーンを活用して、読みやすく効率的なコードを書くことが目標です。
要件
以下の3つのメソッドを実装してください：

total_completed_amount(orders)

ステータスが :completed の注文の合計金額を計算
select と sum をメソッドチェーンで繋げて実装


high_value_customer_names(orders, threshold)

指定金額以上の注文をした顧客名の一覧を返す（重複なし）
select, map, uniq をメソッドチェーンで繋げて実装


average_completed_order_amount(orders)

完了した注文の平均金額を計算（小数点以下も含む）
完了した注文がない場合は 0.0 を返す
select, map, sum, size などを組み合わせて実装



テストケース
rubyorders = [
  { customer: "田中", amount: 15000, status: :completed },
  { customer: "佐藤", amount: 8000, status: :pending },
  { customer: "鈴木", amount: 25000, status: :completed },
  { customer: "田中", amount: 12000, status: :completed },
  { customer: "高橋", amount: 30000, status: :completed },
  { customer: "佐藤", amount: 5000, status: :cancelled },
  { customer: "鈴木", amount: 18000, status: :pending }
]

# total_completed_amount のテスト
total_completed_amount(orders)
# => 82000  (15000 + 25000 + 12000 + 30000)

# high_value_customer_names のテスト
high_value_customer_names(orders, 20000)
# => ["鈴木", "高橋"]

high_value_customer_names(orders, 10000)
# => ["田中", "鈴木", "高橋"]

# average_completed_order_amount のテスト
average_completed_order_amount(orders)
# => 20500.0  (82000 / 4)

orders_no_completed = [
  { customer: "佐藤", amount: 8000, status: :pending },
  { customer: "佐藤", amount: 5000, status: :cancelled }
]

average_completed_order_amount(orders_no_completed)
# => 0.0
ヒント

メソッドチェーン: 複数のメソッドを . で繋げて書く

ruby  array
    .select { |item| condition }
    .map { |item| transform }
    .sum

uniq: 配列から重複を削除
ステータスのチェック: order[:status] == :completed
空配列チェック: empty? または size.zero?
