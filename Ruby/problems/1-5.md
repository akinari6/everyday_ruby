問題2-2: ハッシュ操作とデータアクセス
課題内容
ユーザー情報のハッシュから、安全にデータを取得・操作するメソッドを実装してください。
要件
以下の3つのメソッドを実装してください：

get_user_name(user)

ユーザーハッシュから名前を取得
:name キーが存在しない場合は "名前未設定" を返す
fetch メソッドを使って実装


get_nested_value(data)

ネストしたハッシュから値を安全に取得
data[:user][:profile][:email] のパスで値を取得
パスの途中が nil でもエラーにならないように実装
dig メソッドを使って実装


add_tag(user, tag)

ユーザーハッシュの :tags 配列に新しいタグを追加
:tags キーが存在しない場合は、新しく配列を作成して追加
||= 演算子を使って実装
元のハッシュを変更する（破壊的メソッド）



テストケース
ruby# get_user_name のテスト
user1 = { name: "田中太郎", age: 25 }
get_user_name(user1)
# => "田中太郎"

user2 = { age: 30 }
get_user_name(user2)
# => "名前未設定"

# get_nested_value のテスト
data1 = { user: { profile: { email: "test@example.com" } } }
get_nested_value(data1)
# => "test@example.com"

data2 = { user: { profile: {} } }
get_nested_value(data2)
# => nil

data3 = { user: nil }
get_nested_value(data3)
# => nil

# add_tag のテスト
user3 = { name: "佐藤", tags: ["ruby"] }
add_tag(user3, "rails")
user3
# => { name: "佐藤", tags: ["ruby", "rails"] }

user4 = { name: "鈴木" }
add_tag(user4, "beginner")
user4
# => { name: "鈴木", tags: ["beginner"] }
ヒント

fetch: hash.fetch(key, default) でデフォルト値を指定できます
dig: hash.dig(:key1, :key2, :key3) で安全にネストした値を取得
||= 演算子: variable ||= value は「変数が nil または false なら値を代入」
配列への追加は << または push を使います
