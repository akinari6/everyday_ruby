## テストケース

### 基本動作
- [ ] 異なる頭文字でグループ化できる
  - 入力: ["apple", "banana", "cherry"]
  - 期待値: {"A"=>["apple"], "B"=>["banana"], "C"=>["cherry"]}

- [ ] 同じ頭文字の文字列をまとめられる
  - 入力: ["apple", "avocado", "apricot"]
  - 期待値: {"A"=>["apple", "avocado", "apricot"]}

### 大文字小文字の扱い
- [ ] 小文字始まりでも大文字キーでグループ化
  - 入力: ["apple", "banana"]
  - 期待値: {"A"=>["apple"], "B"=>["banana"]}

- [ ] 大文字小文字混在
  - 入力: ["Ruby", "rails", "React"]
  - 期待値: {"R"=>["Ruby", "rails", "React"]}

### エッジケース
- [ ] 空配列
  - 入力: []
  - 期待値: {}

- [ ] 空文字列を除外
  - 入力: ["test", "", "data"]
  - 期待値: {"T"=>["test"], "D"=>["data"]}

- [ ] 空文字列のみ
  - 入力: ["", ""]
  - 期待値: {}
